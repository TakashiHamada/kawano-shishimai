# 音声エディタ機能 実装プラン

## 概要
現在の単一プレイヤーを、2つの音声トラックを同時に開き、波形上で範囲選択・コピー・ペースト（挿入）・ダウンロード保存ができる音声エディタに拡張する。

## アーキテクチャ

### 技術スタック（追加ライブラリなし、すべて標準Web API）
- **Web Audio API**: 音声デコード・再生・バッファ操作
- **Canvas API**: 波形描画・範囲選択UI
- **WAV形式でエクスポート**: ブラウザ内でエンコード可能（MP3エンコードはライブラリが必要なため避ける）

### ページ構成
現在の `index.html` 単一ファイル構成を維持。ただし、画面が大幅に変わるため **編集モード** を新設し、既存のプレイヤー画面と切り替える形にする。

```
[通常モード] ←→ [編集モード]
  既存プレイヤー        音声エディタ
```

## UI設計

### 編集モード画面レイアウト
```
┌─────────────────────────────────┐
│  ← プレイヤーに戻る            │
│                                 │
│  ── トラック A ──              │
│  [ファイル選択ドロップダウン]    │
│  [=====波形Canvas=====]         │
│  [▶再生] [範囲: 00:15→00:45]   │
│  [コピー]                       │
│                                 │
│  ── トラック B ──              │
│  [ファイル選択ドロップダウン]    │
│  [=====波形Canvas=====]         │
│  [▶再生] [挿入位置: 01:30]      │
│  [ここにペースト] [ダウンロード] │
│                                 │
│  ステータス: コピー済み(30秒)   │
└─────────────────────────────────┘
```

### 操作フロー
1. トラックAで音声ファイルを選択 → 波形が表示される
2. 波形上をドラッグして範囲選択（選択範囲がハイライト表示）
3. 「コピー」ボタンを押す → 選択範囲のAudioBufferをクリップボードに保持
4. トラックBで別の音声ファイルを選択 → 波形が表示される
5. 波形上をクリックして挿入位置を指定（縦線で表示）
6. 「ここにペースト」ボタンを押す → 挿入位置にコピー内容が挿入される
7. トラックBの波形が更新され、挿入結果を再生して確認できる
8. 「ダウンロード」ボタンでWAVファイルとして保存

## 実装ステップ

### Step 1: 編集モードの基本UI
- 「編集モード」切り替えボタンを既存プレイヤーに追加
- 編集モードのHTML/CSS骨組みを作成（2トラック構成）
- トラックごとにファイル選択ドロップダウンを配置

### Step 2: Web Audio APIによる音声デコードとバッファ管理
- AudioContextの初期化
- 選択されたMP3ファイルをfetchしてAudioBufferにデコード
- トラックごとにAudioBufferを保持する状態管理

### Step 3: Canvas波形描画
- AudioBufferのデータからCanvas上に波形を描画する関数
- 描画はダウンサンプリングして適切な解像度に
- トラックごとに独立したCanvasを使用

### Step 4: 波形上での範囲選択（トラックA）
- マウスドラッグ/タッチで範囲選択
- 選択範囲をCanvas上にハイライト表示
- 選択範囲の開始・終了時間を表示

### Step 5: 波形上での挿入位置指定（トラックB）
- クリック/タップで挿入位置を指定
- 挿入位置を縦線で表示
- 挿入位置の時間を表示

### Step 6: コピー＆ペースト機能
- 「コピー」: トラックAの選択範囲からAudioBufferを切り出して保持
- 「ペースト」: トラックBのAudioBufferの指定位置にコピー内容を挿入
  - 新しいAudioBufferを作成（元の長さ + コピー部分の長さ）
  - 挿入位置前 → コピー内容 → 挿入位置後 の順にデータをコピー
- ペースト後、トラックBの波形を再描画

### Step 7: 再生機能（編集モード用）
- 各トラックの再生/停止ボタン
- 再生位置をCanvas上に縦線で表示（requestAnimationFrame）
- 選択範囲のみ再生する機能

### Step 8: WAVダウンロード機能
- AudioBufferからWAVフォーマットのバイナリを生成
- Blobを作成してダウンロードリンクを生成
- ファイル名は「編集済み_[元のファイル名].wav」

### Step 9: UIの仕上げとモバイル対応
- 編集モードはmax-width制限を解除（幅広く使う）
- タッチ操作での範囲選択・位置指定の動作確認
- 横向きでの使用も検討（波形が見やすい）
- ステータスメッセージの表示

## 注意点
- 大きなMP3ファイルのデコードには時間がかかる → ローディング表示が必要
- AudioBufferはメモリを消費する → 同時に2トラック分のみ保持
- WAVファイルは非圧縮のため大きくなる → ダウンロード時に注意表示
- チャンネル数・サンプルレートの違いへの対処（リサンプリング）
